# Joint inversion of dike opening using seismicity and fibre-optic geodesy
Dike intrusions are commonly interpreted using both geodetic and seismic observations, but the two datasets are rarely combined into a single quantitative model. This thesis develops a joint inversion framework that uses \ac{LFDAS}, GNSS and multiple earthquake catalogues to constrain dike opening during the recent Sundhnúkagígar intrusion sequence in Iceland. The geodetic part of the method uses a linear inversion for dike opening and Mogi-source deflation. The seismicity part estimates earthquake-rate histories in spatial voxels with a trapezoidal approximation and converts them to stress changes using a revised Dieterich 1994 law. The aim is not to recover a unique or fully physics-based propagation model, but to use seismicity to condition a non-unique geodetic inversion so that the final opening model remains consistent with both the deformation data and the observed earthquake distribution. Synthetic tests show that adding seismicity sharpens the recovered dike image and improves recovery of the true model relative to a DAS-only inversion. Application to the real data, especially on the eruption on the 14 January 2024, shows that seismicity helps constrain the extent and position of the modelled intrusion while preserving consistency with the DAS observations. The results suggest that earthquake locations can provide a useful additional constraint on dike-opening models derived from fibre-optic geodesy. The method presented is best viewed as a practical conditioning tool for rapid geodetic inversions and as a step toward more quantitative joint analysis of deformation and seismicity during volcanic unrest.

## Structure of the repository
This repository contains:
- The PDF of the thesis [MS_thesis_Schindfessel.pdf](MS_thesis_Schindfessel.pdf)
- A folder with all the codes used in the thesis: [Codes](Codes/). Coding was carried out with MATLAB R2023a.
- A folder with the figures shown in the thesis: [Figures](Figures/).
- A folder with the movies of the different models: [Movies](Movies/). They are named following a simple structure: First the event number (E1, E2, ...), followed by the datasets used for the inversion.
